# nuclear_simu

核が細胞質から脱落する条件を、FEBio を主系にして評価するための研究用フロントエンドです。

## Read First

1. `AGENT.md`
2. `CODEBASE_STRUCTURE.md`
3. relevant `.skills/**/SKILL.md`
4. `PROGRESS.md`

## Current Main Path

- UI input
- canonical spec
- FEBio template / `.feb` XML
- FEBio CLI execution
- normalized result import
- physical result rendering

## Source Of Truth

- parameter schema: `src/model/schema.ts`
- mesh generation: `src/febio/mesh/`
- nucleus-cytoplasm interface: `src/febio/interfaces/nucleusCytoplasm.ts`
- FEBio export: `src/febio/export/index.ts`
- FEBio export detachment contract: `src/febio/export/index.ts`
  export bundles declare explicit detachment event and metric ownership for external FEBio results.
- result normalization: `src/febio/import/normalizeFebioResult.ts`
  native-first `localNc` normalization, native face-data fallback, explicit external detachment event normalization, detachment metric supplementation, and imported event backfill.
- classification: `src/results/classification.ts`
  native-first detachment judgment with explicit proxy fallback labeling.
- public detachment assessment: `src/public-api.ts`
  exposes canonical `assessDetachment` so compatibility/runtime code can reuse the same thresholds.
- nucleus-cytoplasm interface stabilization: `src/febio/interfaces/nucleusCytoplasm.ts`
  solver-primary sticky cohesive approximation, explicit soft-start stabilization ramp, and proxy/native observation ownership.

## Compatibility Note

- `src/` is the canonical edit target.
- `dist/` is generated by `node scripts/build-dist.mjs`.
- `index.html` now loads `dist/browser/main.js`.
- The current browser runtime still boots the legacy compatibility bundle (`simulation.js`, `js/simulation-febio.js`, `js/simulation-ui.js`) through that compiled entry while the source-of-truth modules are moved into `src/`.
- Classification callers in the legacy runtime now prefer the canonical public API classifier bridge before falling back to compatibility-local logic.
- Lightweight compatibility runtime now emits explicit `detachmentStart` / `detachmentComplete` events using canonical public API detachment assessment when available.

## Commands

- build: `node scripts/build-dist.mjs`
- source parse check: `node --experimental-strip-types -e "import('./src/public-api.ts')"`
- tests: `node --test tests/*.test.mjs`
- npm wrapper: `cmd /c npm.cmd test`

## Docs Layout

- keep only `AGENT.md`, `README.md`, `PROGRESS.md`, and `CODEBASE_STRUCTURE.md` at the repo root
- FEBio design / mapping / bridge notes live under `docs/febio/`
- research model notes live under `docs/research/`
- generated FEBio export artifacts should not stay at the repo root

## Key Rules

- Edit canonical logic in `src/`, not in generated `dist/`.
- When changing physics model, cohesive behavior, detachment logic, main flow, classification, or export/import ownership, update `PROGRESS.md` in the same change set.
- Keep `README.md`, `CODEBASE_STRUCTURE.md`, and `PROGRESS.md` aligned.
