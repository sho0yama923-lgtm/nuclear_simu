## Active / Legacy FEBio Ownership

Active FEBio export work now lives only in:

- `febio_cases/native/*.native.json`
- `src/febio/native/`
- `scripts/export_febio_native_case.mjs`
- `tests/febio-native-pipeline.test.mjs`
- `febio_exports/<case>/` generated by that script
- `docs/febio/GEOMETRY_CONVENTIONS.md`

All previous FEBio export, UI/canonical conversion, public API, browser bridge, import, and classification paths are legacy / compatibility for this purpose. Retired docs and historical generated artifacts are under [legacy/](../legacy/). See [docs/febio/FEBIO_PATH_OWNERSHIP.md](febio/FEBIO_PATH_OWNERSHIP.md).

## 正規情報源の配置

- `febio_cases/native/*.native.json`
  - active FEBio case condition values の source-of-truth。
- `src/febio/native/`
  - active native-only FEBio model expansion、XML serialization、export bundle assembly の source-of-truth。
- `scripts/export_febio_native_case.mjs`
  - active native-only FEBio handoff artifact writer。
- `src/model/schema.ts`
  - legacy / compatibility parameter schema、validation、digest、schedule metadata、normalized input spec。
- `src/model/defaults.ts`
  - editable field groups と default values。
- `src/model/types.ts`
  - shared region names、schema constants、coordinate metadata。
- `src/febio/mesh/index.ts`
  - mesh と surface-pair の source-of-truth。native direct path は現時点で `s7-debug-local-nucleus` の粗い確認用 mesh を明示し、pipette suction surface を nucleus-side capture face へ置く。
- `src/febio/spec/index.ts`
  - legacy / compatibility FEBio-native direct template mapping。新しい active FEBio output path では触らない。
- `src/febio/interfaces/nucleusCytoplasm.ts`
  - nucleus-cytoplasm interface の source-of-truth。sticky cohesive stabilization、stabilization validation、proxy/native observation ownership を含む。
- `src/febio/export/index.ts`
  - legacy / compatibility FEBio template assembly、XML serialization、export bundle entrypoint、detachment event contract metadata、face-data coverage metadata、plotfile `contact traction` bridge contract。
- `src/febio/import/normalizeFebioResult.ts`
  - legacy / compatibility FEBio result normalization entrypoint。active export path では呼ばない。
- `src/results/classification.ts`
  - legacy / compatibility classification と detachment interpretation。active export path では呼ばない。
- `src/public-api.ts`
  - legacy / compatibility public API。active native-only export path では呼ばない。
- `src/browser/main.ts`
  - browser entry。public API を公開し、legacy compatibility scripts を起動する。

## 生成物と派生物

- `generated/dist/`
  - `src/` から `node scripts/build-dist.mjs` で生成される。手で編集しない。
- `generated/febio_exports/`
  - FEBio export bundle、bridge output、scan、historical run artifacts の既定出力先。
- `tmp/logs/`
  - bridge output logs などの disposable local logs。

## 互換バンドル

- `simulation.js`
  - browser app 用 legacy compatibility layer。schema / classification / detachment の長期 source-of-truth ではない。
- `js/simulation-febio.js`
  - legacy FEBio compatibility layer。
- `js/simulation-ui.js`
  - legacy UI runtime。

Compatibility は退役経路に置く。新しい長期責務は canonical `src/` module に置き、compatibility 側は bridge/caller として薄くする。

## ビルドと入口

- `scripts/build-dist.mjs`
  - `src/**/*.ts` を `generated/dist/**/*.js` へコピーし、relative import extensions を書き換える。
- `scripts/export_febio_case.mjs`
  - legacy / compatibility canonical UI parameter path から FEBio handoff bundle を生成する。新しい solver behavior では触らない。
- `scripts/export_febio_direct_case.mjs`
  - legacy / compatibility S7-A/B/C direct FEBio handoff bundle generator。新しい solver behavior では触らない。
- `scripts/export_febio_native_case.mjs`
  - active native-only FEBio handoff bundle generator。`src/febio/native/index.ts` だけを入口にする。
- `scripts/convert_febio_output.mjs`
  - legacy / compatibility FEBio logfile output converter。active export path では manifest に expected artifacts として提示するだけで呼ばない。
- `index.html`
  - `generated/dist/browser/main.js` を読み、legacy browser scripts を順に起動する。

## 探索時の注意

- source-of-truth は `src/` を優先する。
- skill や AGENT rule が対象ファイルを示している場合、repo-wide exploration は避ける。
- public `index.ts` がある場合、internal file への deep import は避ける。
- `generated/` と `tmp/` は durable source ではない。
- 2026-04-24 note: FEBio export / conversion scripts use `src/public-api.ts` directly and do not read legacy JS simulation files. Browser compatibility scripts remain a UI bridge only; they are not source-of-truth for classification, detachment, export, or import.
